# LovelyGit frontend

This directory contains the React 19, TypeScript, Tailwind v4, and Vite frontend
for LovelyGit. The ASP.NET Core desktop host serves the built assets from
`LovelyGit/wwwroot`, and the WebView2 app loads that production output.

## Setup

Install dependencies from this directory:

```powershell
pnpm install --frozen-lockfile
```

The repository pins `pnpm@11.3.0` through `packageManager` in `package.json`.

## Common commands

Run these commands from `LovelyGit/Frontend` unless noted otherwise:

| Command | Purpose |
| --- | --- |
| `pnpm dev` | Start Vite for browser-only frontend development. |
| `pnpm watch` | Regenerate contracts, then watch backend native-message contracts. |
| `pnpm generate:csharp-types` | Regenerate TypeSharp output and native-message contract types. |
| `pnpm check:types` | Regenerate types and fail if generated files changed. |
| `pnpm build` | Run codegen, TypeScript build, and Vite production build. |
| `pnpm prod` | Alias for the production build used before backend packaging. |
| `pnpm test` | Run the Vitest suite once. |
| `pnpm lint` | Run Biome checks with writes enabled. |
| `pnpm lint:fix` | Same Biome write-enabled check as `pnpm lint`. |

`vite.config.ts` writes production output directly to `../wwwroot`, so a
frontend production build refreshes the static files served by `LovelyGit`.

## Generated contracts

Do not edit files under `src/generated` manually. They are generated by:

- TypeSharp from backend C# types using `typesharp.config.json`.
- `scripts/generate-native-message-contracts.mjs` from
  `Services/NativeMessaging/NativeMessageType.cs`.

When backend command, setting, or native-message contracts change, run
`pnpm generate:csharp-types` and include the generated diff with the backend
change.

## Source layout

- `src/main.tsx` starts `bootstrapApp`.
- `src/bootstrapApp.tsx` initializes settings, applies theme and font values,
  then renders React.
- `src/App.tsx` switches between the new-tab experience and repository views.
- `src/components` contains feature UI and shared shadcn-style primitives.
- `src/lib` contains SignalR/native-message clients, settings, repository
  context, theme/font helpers, and general utilities.
- `src/generated` contains generated TypeScript contracts.

## Development notes

Use the `@/*` import alias for code under `src`; it is configured in both Vite
and TypeScript. React Compiler is enabled in `vite.config.ts` through
`@vitejs/plugin-react` and `@rolldown/plugin-babel`, so avoid adding manual
memoization unless a measured hotspot or existing local pattern calls for it.
